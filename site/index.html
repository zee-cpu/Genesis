Genesis
A local-first governed business operating engine.

Opportunities in. Bounded experiments, signed decisions, measured outcomes, and reusable learning out.

Genesis is a Node.js CLI that governs how a business idea becomes a real, bounded experiment: evidence is preserved with its counterevidence, every experiment is preregistered with hard limits, approval requires an SSH-signed decision from a named human, and everything lands in an immutable, auditable record set on your own disk.

Core capabilities

Discipline you can audit. Privacy you don't have to trust.

Guided CLI workflow

genesis next explains the current state and walks the one valid transition, one question at a time — with a preview and explicit confirmation before anything is written.

SSH-verified Human Authority

Approvals are signed with the append-only genesis-owner identity key and verified on every use. Unsigned text grants no authority.

Immutable YAML records

Decisions, evidence, experiments, and approvals are append-only, schema-validated files. Corrections create new versions; history is never rewritten.

Conflict-safe Git event syncing

Records travel as content-addressed events in your normal Git workflow. Identical events merge by union; true conflicts stop and escalate to a human — never auto-resolve.

Rebuildable SQLite projection

Fast queries come from a derived local index that can always be discarded and rebuilt from canonical YAML with genesis rebuild-index.

Human approval gates

Protected transitions require an explicit, exactly-scoped decision by the Human Authority — with actor, limits, effective time, and expiry checked on every use.

Evidence and counterevidence

Contradicting evidence is captured with source and provenance as first-class data — beliefs are supposed to change when reality does.

Measurement and reflection

Experiments close with actual cost, observed results, data-quality limits, a reflection, a confidence update, and a reusable lesson — not vibes.

Read-only Operator Console

A local web console at localhost:3000 visualizes your records, approvals, and sync health. It can copy CLI commands — it can never run them.

Boundaries by design

What Genesis deliberately does not do

—

No autonomous execution. Genesis does not run businesses, contact customers, or deploy products. Humans execute; Genesis governs and records.

—

No inferred approval. Silence, authorship, and past behavior grant nothing. Only a valid, signed, unexpired Human Authority record authorizes a protected action.

—

No private keys stored. Signing uses your SSH key in place; Genesis records only public keys, fingerprints, and verification results.

—

SQLite is derived. The index is a rebuildable cache. Canonical YAML remains the single source of truth.

—

The web console is read-only. It binds to 127.0.0.1, rejects every mutation method, and suggests CLI commands instead of running them.

—

Offline by default. Normal operation makes zero network requests. Syncing rides your own Git workflow; Genesis never touches a remote.
